## Phase 1 — 상관관계 분석 (포트폴리오 심화)

포트폴리오 페이지 기능 확장.

- [x] **상관관계 히트맵** — 포트폴리오에 담은 ETF 간 상관계수 매트릭스 시각화
  - 프론트: `CorrelationHeatmap.tsx` 컴포넌트 (색상 스케일: -1 파랑 ~ 0 회색 ~ +1 빨강)
  - 평시/꼬리 상관 토글 지원
- [x] **분산 효과 진단** — "진짜 분산 vs 가짜 분산" 경고
  - 높은 상관 쌍 감지 (≥0.85), 평균 상관 진단, 음의 상관 존재 여부
  - 위기 시 동조 위험 경고 (꼬리 상관 급등 감지)
- [x] **꼬리 상관관계** — 하락 구간(하위 5%) 조건부 상관계수
  - 평시/꼬리 탭 전환, 꼬리 상관 급등 시 자동 경고

### 필요한 백엔드 작업
- [ ] `POST /api/portfolio/correlation` — 티커 배열 → 상관 매트릭스 + 꼬리 상관 매트릭스 반환
  - 요청: `{ tickers: string[] }`
  - 응답: `{ tickers: string[], matrix: number[][], tail_matrix: number[][] }`
- [ ] `POST /api/portfolio/analyze` — individual 배열에 배당 포함 필드 추가
  - 현재: `annual_return`, `annual_volatility`, `sharpe_ratio`만 반환
  - 추가 필요: `total_annual_return`, `total_sharpe_ratio` (포트폴리오 합계에서는 이미 계산 중, 개별 ETF에도 동일 로직 적용)
  - 프론트는 반영 완료 — 백엔드에서 내려주면 바로 표시됨

## Phase 2 — 매크로 레짐 인디케이터

Market 페이지 기능 확장.

- [ ] **레짐 분류 위젯** — 2×2 매트릭스(성장×인플레) 위에 현재 위치 표시

## Phase 3 — 레짐-적응형 포트폴리오

포트폴리오 + 매크로 레짐을 연결.

- [ ] **레짐 연동 백테스트** — 과거 레짐 구간별 포트폴리오 성과 분해
  - "골디락스 구간에서의 수익률" vs "스태그플레이션 구간에서의 수익률"
- [ ] **코어-새틀라이트 빌더** — 코어/새틀라이트/꼬리보호 3계층 포트폴리오 구성 UI
  - 각 계층의 역할 설명 + 비중 가이드
- [ ] **리밸런싱 알림 규칙** — 레짐 전환 신호 감지 시 리밸런싱 제안
  - 수익률곡선, 신용 스프레드, VIX 등 트리거 조건 설정

### 필요한 백엔드 작업
- 과거 레짐 구간 라벨링 데이터
- `POST /api/portfolio/regime-analysis` — 레짐별 성과 분해

## 우선순위 판단 기준

| 기준           | Phase 1 | Phase 2 | Phase 3 | Phase 4 |
|---------------|---------|---------|---------|---------|
| 사용자 가치     | 높음    | 높음    | 중간    | 중간    |
| 구현 난이도     | 낮음    | 중간    | 높음    | 중간    |
| 백엔드 의존도   | 중간    | 높음    | 높음    | 중간    |
| 기존 코드 활용  | 높음    | 중간    | 낮음    | 높음    |

**추천 순서: Phase 1 → Phase 4 → Phase 2 → Phase 3**
- Phase 1은 기존 포트폴리오 페이지에 바로 붙일 수 있고, 백엔드 계산도 비교적 단순
- Phase 4는 기존 비교 페이지 확장이라 코드 활용도가 높음
- Phase 2, 3은 매크로 데이터 수집 인프라가 필요해서 후순위
